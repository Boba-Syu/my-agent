# ReAct 模式指导文档

## 什么是ReAct

ReAct（Reasoning + Acting）是一种将推理和行动结合的智能体模式。智能体通过交替进行"思考"和"行动"来解决复杂任务。

## ReAct循环

```
┌─────────────┐
│   Thought   │ ← 思考当前状态和目标
└──────┬──────┘
       ↓
┌─────────────┐
│   Action    │ ← 决定执行什么操作
└──────┬──────┘
       ↓
┌─────────────┐
│ Observation │ ← 观察行动结果
└──────┬──────┘
       ↓
   [循环或结束]
```

## 在RAG中的应用

### 标准RAG流程

```
用户问题 → 检索 → 生成答案
```

### Agentic RAG流程（ReAct）

```
用户问题 → [思考] → 决策 → [行动] → 观察 → [思考] → ... → 答案
```

## 决策点

### 1. 是否需要检索？

**需要检索的信号：**
- 问题涉及具体知识领域
- 你不确定答案
- 需要最新或详细信息
- 问题涉及事实性内容

**不需要检索的信号：**
- 闲聊、问候
- 明确的能力范围外
- 简单的事实（如"你叫什么"）

### 2. 检索后的判断

**结果充足：**
- 有多个高相关度文档
- 内容可直接回答问题
- 进入答案生成阶段

**结果不足：**
- 无相关文档
- 内容不完整
- 可能需要：
  - 换关键词重新检索
  - 告知用户无相关信息
  - 基于常识给出建议

### 3. 是否需要多轮检索？

**多轮场景：**
- 问题涉及多个子话题
- 首次检索结果不全面
- 用户追问更细节内容

## 工具调用规范

### hybrid_search（混合检索）

```
思考：用户询问"如何创建工作流"，需要检索知识库
行动：调用hybrid_search
参数：{
  "query": "创建工作流步骤",
  "kb_types": ["faq"],
  "top_k": 10
}
观察：返回5条相关文档...
```

### get_context（获取上下文）

```
思考：检索结果较多，需要精排序
行动：调用get_context
参数：{
  "query": "用户原始问题",
  "search_results": [...],
  "top_k": 5
}
观察：返回精选的3条高质量上下文...
```

### generate_answer（生成答案）

```
思考：已获取上下文，可以生成答案
行动：调用generate_answer
参数：{
  "query": "用户问题",
  "context": "精选上下文"
}
观察：返回完整的答案...
```

## 提示词模板

### 基础ReAct模板

```
请按照ReAct模式解决问题：

问题：{question}

步骤：
1. Thought: 分析问题，决定是否需要检索
2. Action: 如果需要，调用hybrid_search
3. Observation: 查看检索结果
4. Thought: 判断结果是否充足
5. Action: 如需要，调用get_context精排序
6. Observation: 查看精选上下文
7. Thought: 准备生成答案
8. Action: 调用generate_answer或基于上下文回答
9. Final Answer: 输出最终答案
```

### 简化模板（直接回答）

```
问题：{question}

分析：这是一个[类型]问题，[不需要/需要]检索。

[如果不需要检索，直接回答]
[如果需要检索，说明检索策略]
```

## 最佳实践

1. **思考要显式**：让模型明确输出思考过程
2. **决策要明确**：是否检索要果断决定
3. **迭代要控制**：设置最大迭代次数防止无限循环
4. **结果要验证**：检索后验证是否真正回答了问题
5. **失败要处理**：无结果时给出合理回应

## 常见模式

### 模式1：直接回答
```
Thought: 这是问候语，不需要检索
Final Answer: 您好！我是...
```

### 模式2：单次检索
```
Thought: 需要查找功能说明
Action: hybrid_search
Observation: 找到相关文档
Thought: 结果充足，直接回答
Final Answer: 根据知识库...
```

### 模式3：检索+精排
```
Thought: 需要深入检索
Action: hybrid_search
Observation: 返回10条结果
Thought: 需要精排序
Action: get_context
Observation: 返回5条精选
Action: generate_answer
Final Answer: 完整答案
```

### 模式4：无结果处理
```
Thought: 需要查找信息
Action: hybrid_search
Observation: 未找到相关文档
Thought: 知识库无覆盖，基于常识回答
Final Answer: 抱歉，知识库中...
```
